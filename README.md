# 商业保险智能问答Agent

基于 LangChain + LlamaIndex 深度整合的商业保险智能问答系统，专为保险业务场景设计，实现混合检索、多Agent协作、智能路由、产品对比、理赔推荐和风险评估的完整流程。

## ✨ 功能特性

- **混合检索引擎**：向量检索 + BM25关键词检索 + RRF融合算法，提升召回率
- **查询智能扩展**：保险专业查询词扩展，生成多维度查询
- **多Agent协作**：咨询Agent + 对比Agent + 理赔Agent + 风险评估Agent
- **智能路由**：自动识别问题类型（咨询/对比/理赔/风险评估/条款解读）
- **产品对比**：多维度保险产品对比分析（保障范围/保费/理赔/免责条款）
- **理赔推荐**：根据事故场景智能推荐理赔方案
- **风险评估**：企业风险等级评估与保险配置建议
- **条款解读**：专业术语解释、免责条款提取
- **多轮对话记忆**：支持上下文对话，保持对话连贯性
- **质量自动评估**：从准确性、完整性、专业性三维度评估答案质量
- **分析报告生成**：自动生成结构化保险分析报告
- **邮件发送**：模拟邮件发送功能，支持报告分发

## 💡 为什么选择 LangChain + LlamaIndex 组合？

### 单独使用的局限性

| 框架 | 优势 | 局限性 |
|------|------|--------|
| **仅使用 LangChain** | 强大的Agent编排、工具调用、链式组合 | 文档处理能力弱，向量检索功能基础，缺乏高级检索策略 |
| **仅使用 LlamaIndex** | 优秀的文档加载、分块、索引构建、高级检索 | Agent系统薄弱，工具调用能力有限，流程编排不够灵活 |

### 组合使用的核心优势

#### 1. **职责分离，各司其职**

```
LlamaIndex 专注「数据层」          LangChain 专注「编排层」
├── 文档加载与解析                ├── Agent系统与工具调用
├── 智能分块与向量化              ├── LCEL链式编排
├── 向量索引构建                  ├── 对话记忆管理
├── 混合检索（向量+BM25）         ├── 智能路由与条件分支
└── 查询转换与后处理              └── 输出解析与格式化
```

#### 2. **检索能力大幅提升**

| 检索特性 | 仅LangChain | 仅LlamaIndex | 组合使用 |
|---------|------------|-------------|---------|
| 向量检索 | ✅ 基础 | ✅ 高级 | ✅ LlamaIndex提供 |
| BM25关键词检索 | ❌ 不支持 | ✅ 支持 | ✅ LlamaIndex提供 |
| RRF融合算法 | ❌ 不支持 | ✅ 支持 | ✅ LlamaIndex提供 |
| 查询扩展 | ❌ 需自建 | ✅ 内置 | ✅ LlamaIndex提供 |
| 检索结果处理 | 基础 | 高级后处理器 | ✅ LlamaIndex提供 |
| 工具化封装 | ✅ 强 | ❌ 弱 | ✅ LangChain提供 |

#### 3. **Agent系统更强大**

- **LangChain的Agent优势**：
  - 成熟的工具调用机制（@tool装饰器）
  - 多种Agent类型（ReAct、Tool Calling、Plan-and-Execute）
  - 强大的对话记忆系统（短期/长期/向量记忆）
  - 灵活的链式编排（LCEL语法）

- **LlamaIndex的检索优势**：
  - 将高级检索能力封装为LangChain工具
  - 检索结果自动转换为LangChain Document格式
  - 支持多路召回、重排序、过滤等复杂策略

#### 4. **1+1>2 的协同效应**

```python
# 示例：LlamaIndex检索 + LangChain工具调用

# LlamaIndex 负责高效检索
class InsuranceRetriever:
    def retrieve(self, query):
        # 混合检索：向量 + BM25 + RRF融合
        vector_results = self.vector_search(query)
        bm25_results = self.bm25_search(query)
        return self.reciprocal_rank_fusion(vector_results, bm25_results)

# LangChain 负责工具化封装
@tool
def search_insurance(query: str) -> str:
    """保险文档检索工具"""
    retriever = InsuranceRetriever()  # LlamaIndex检索器
    results = retriever.retrieve(query)
    return format_results(results)  # LangChain格式输出

# LangChain Agent 自动调用工具
agent = create_tool_calling_agent(llm, tools=[search_insurance, ...])
```

## 🔧 核心功能

### 1. 高级检索技术

- **混合检索**：向量检索（语义相似度）+ BM25（关键词匹配）双路召回
- **RRF融合**：Reciprocal Rank Fusion算法融合多路检索结果
- **查询扩展**：保险专业查询词扩展，自动生成相关查询
- **产品过滤**：支持按保险产品名称过滤检索结果
- **相似度过滤**：自动过滤低相关度结果

### 2. 多Agent协作

| Agent | 功能 | 触发场景 |
|-------|------|---------|
| 咨询Agent | 保险产品咨询解答 | 保障范围、保费、条款咨询 |
| 对比Agent | 多产品对比分析 | 产品优劣势对比 |
| 理赔Agent | 理赔方案推荐 | 理赔流程、材料、条件 |
| 风险评估Agent | 企业风险评估 | 风险等级、保险配置建议 |
| 条款解读Agent | 专业术语解释 | 术语解释、免责条款 |

### 3. 智能路由系统

自动识别问题类型，选择对应处理链：

- `CONSULTATION` → 咨询问答链
- `COMPARISON` → 产品对比链
- `CLAIM` → 理赔推荐链
- `RISK_ASSESSMENT` → 风险评估链
- `POLICY_INTERPRETATION` → 条款解读链

### 4. 对话记忆系统

- **上下文管理**：保存最近5轮对话历史
- **连贯性保持**：在生成回答时融入对话历史
- **灵活控制**：支持查看、清空对话历史

### 5. 质量评估机制

- **准确性评估**（40分）：答案是否准确反映保险条款
- **完整性评估**（30分）：答案是否全面覆盖问题要点
- **专业性评估**（30分）：是否使用专业术语和表述

### 6. 保险专属工具

| 工具 | 功能 |
|------|------|
| search_insurance | 保险文档检索 |
| compare_insurance_products | 保险产品对比 |
| get_claim_guide | 理赔指引 |
| explain_term | 保险术语解释 |
| save_report | 报告保存 |
| send_email | 邮件发送 |
